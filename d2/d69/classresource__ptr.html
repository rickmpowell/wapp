Detailed Description

a managed pointer to a resource object in the application's resource fork.

A pointer to a resource object in the application's resource fork. Uses similar semantics to unique_ptr.

Resources do not need to be freed in modern Windows, so this implementation is a little weird. Also, the get() function returns a pointer rather than a handle, and reset returns the handle to the data. We do not keep the resource handle internally, so that handle is lost.

The size is lost after a reset(), so use it with care.

REVIEW: should this be a templated class?
